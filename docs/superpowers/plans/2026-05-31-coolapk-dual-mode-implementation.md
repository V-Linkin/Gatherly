# 酷安双模式解析器实现计划

## 项目概述
基于已验证的设计方案，实现酷安双模式解析器，解决当前无法获取正文、图片、作者信息的问题。

## 实现目标
- ✅ 实现 HTTP + WebView 双模式解析架构
- ✅ 提取完整的文章内容、图片列表、作者信息
- ✅ 复用现有基础设施，保持代码一致性
- ✅ 通过测试验证功能完整性

## 阶段划分

### 阶段1：基础架构改造（预计1天）
**目标**: 修改 CoolapkParser 采用双模式架构

#### 1.1 修改 CoolapkParser 结构
- 文件: `Parsers/CoolapkParser.swift`
- 任务:
  - 添加 `parseViaHTTP(url:)` 方法
  - 添加 `parseViaWebView(url:)` 方法
  - 修改 `parse(url:)` 方法实现双模式切换
  - 保持现有的 `downloadMedia` 方法不变

#### 1.2 HTTP 模式实现
- 复用现有 URLSession 配置
- 添加 SSR 数据检测逻辑
- 添加 Meta 标签回退逻辑
- 实现请求间隔控制（类似豆瓣）

#### 1.3 WebView 模式集成
- 集成 `ZhihuWebLoader`
- 实现超时处理和错误降级
- 保持与小红书解析器一致的调用模式

### 阶段2：WebView JavaScript 提取逻辑（预计2天）
**目标**: 实现酷安页面内容提取的 JavaScript 逻辑

#### 2.1 分析酷安页面结构
- 测试不同类型的酷安页面（动态、文章、帖子）
- 确定最佳的选择器策略
- 验证 SSR 数据结构

#### 2.2 实现 JavaScript 提取逻辑
- 文件: `Utilities/ZhihuWebLoader.swift`
- 任务:
  - 添加酷安页面识别逻辑
  - 实现 SSR 数据提取
  - 实现 DOM 结构提取
  - 实现图片过滤逻辑
  - 实现作者信息清洗

#### 2.3 优化提取策略
- 标题提取：避免页面标题干扰
- 图片过滤：去除logo、表情包、头像等
- 作者清洗：处理换行符和设备信息
- 封面选择：使用第一张内容图片

### 阶段3：数据模型适配（预计0.5天）

#### 3.2 媒体下载逻辑
